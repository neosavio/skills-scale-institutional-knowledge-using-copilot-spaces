# OctoAcme Project Management Docs – Overview

Welcome to the collection of OctoAcme's project management process documents. This README provides a summary of our standardized project management approach and links to all process docs for easier navigation.

## OctoAcme Project Management Summary

OctoAcme follows a structured, customer-focused approach to project management:

- **Customer-focused, iterative delivery cycles** — Prioritize customer value and usability through small, testable increments
- **Clear ownership** — Each project has a named Project Manager (PM) and Product Manager (PdM) responsible for coordinated delivery and value definition
- **Data-informed decisions** — Measure impact and iterate based on evidence and key success metrics
- **Structured project lifecycle** — Five key phases: Initiation → Planning → Execution → Release → Retrospective
- **Emphasis on risk tracking, transparency, and continuous improvement** — Proactively identify and mitigate risks while maintaining open communication with stakeholders

## Core Principles

1. **Customer-first** — Prioritize customer value and usability
2. **Iterative delivery** — Deliver small, testable increments
3. **Clear ownership** — Each project has a named PM and Product Lead
4. **Data-informed decisions** — Measure impact and iterate based on evidence
5. **Psychological safety** — Encourage feedback and learning

## Project Lifecycle Overview

| Phase | Purpose | Key Deliverables |
|-------|---------|------------------|
| **Initiation** | Validate business need and authorize work | Project One-pager, Stakeholder alignment, Timeline |
| **Planning** | Turn approved initiative into actionable plan | Prioritized backlog, Definition of Done, Release plan |
| **Execution** | Build, test, review, and iterate | Working software, Test results, Progress metrics |
| **Release** | Deploy to production and verify | Release notes, Rollback plan, Stakeholder announcement |
| **Retrospective** | Capture learnings and improvements | Retrospective notes, Action items, Improvement tracking |

## Core Roles

- **Project Manager (PM)** — Coordinates delivery, manages schedules, risks, and communications
- **Product Manager (PdM)** — Defines outcomes, prioritizes the backlog, measures success
- **Developers** — Implement features, collaborate on design and testability
- **QA Lead** — Oversees quality strategy, test plans, and release-readiness sign-off
- **UX/UI Designer** — Creates wireframes, mockups, and design assets; ensures usability standards
- **Release Manager** — Owns the release workflow and coordinates deployment activities
- **Customer Support Lead** — Bridges end users and the delivery team; feeds back user issues
- **Technical Writer** — Maintains accurate, up-to-date documentation across all project phases
- **Stakeholders** — Provide inputs, approvals, and feedback

For detailed persona definitions, see [Roles and Personas](./octoacme-roles-and-personas.md).

## Key Artifacts & Templates

Common artifacts used across OctoAcme projects:

- **Project Charter / One-pager** — Problem statement, goal, success metrics, team, timeline
- **Risk Register** — Tracking ID, description, impact, likelihood, owner, mitigation
- **Project Board** — Backlog, Ready, In Progress, In Review, QA, Done columns
- **Backlog Items** — Title, description, acceptance criteria, priority, estimate, owner
- **Weekly Status** — Progress, next steps, risks, blockers, decisions needed
- **Retrospective Notes** — What went well, improvements, action items with owners

## Communication Cadence

- **Daily** — Team standups (15 minutes)
- **Weekly** — PM + PdM sync; delivery team status sync
- **Milestone-based** — Stakeholder updates, demos, retrospectives
- **As needed** — Risk escalations and incident communications
