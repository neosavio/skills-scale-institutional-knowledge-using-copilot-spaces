# OctoAcme — New Team Member Onboarding Checklist

## Purpose
Help new team members across all roles quickly orient themselves to OctoAcme's processes, tools, and documentation. This checklist should be completed within the first week of joining the project.

## When to Use
Assign this checklist to every new team member regardless of role. Role-specific sections at the bottom highlight additional steps for each function.

---

## Day 1 — Project Orientation

- [ ] Read `docs/octoacme-project-management-overview.md` for a full project overview
- [ ] Read `docs/octoacme-roles-and-personas.md` and identify your role and key collaborators
- [ ] Review the project's README and repository structure
- [ ] Set up access to required tools (project board, communication channels, code repo)
- [ ] Attend (or watch recording of) the most recent team standup or sync

## Week 1 — Process Familiarisation

- [ ] Review `docs/octoacme-project-initiation.md` to understand how work gets started
- [ ] Review `docs/octoacme-project-planning.md` to understand planning artifacts
- [ ] Review `docs/octoacme-execution-and-tracking.md` for day-to-day delivery practices
- [ ] Review `docs/octoacme-risks-and-communication.md` for risk and escalation procedures
- [ ] Review `docs/octoacme-release-and-deployment.md` for release practices
- [ ] Review `docs/octoacme-retrospective-and-continuous-improvement.md`
- [ ] Schedule 1:1 meetings with your direct collaborators (see role-specific section below)
- [ ] Confirm your availability with the Project Manager for upcoming milestones

---

## Role-Specific Steps

### Developer
- [ ] Clone the repository and verify local development environment setup
- [ ] Review the Definition of Done and coding standards
- [ ] Introduce yourself in the team's communication channel

### Product Manager
- [ ] Review the current roadmap and backlog with the Project Manager
- [ ] Identify the top 3 user personas and current success metrics

### Project Manager
- [ ] Review the current project plan, risk register, and open dependencies
- [ ] Confirm ownership of the project board and status report cadence

### QA Lead
- [ ] Review existing test plans and the current defect log
- [ ] Align with the Release Manager on the release-readiness sign-off process (see `docs/octoacme-release-checklist.md`)
- [ ] Identify gaps in automated test coverage

### UX/UI Designer
- [ ] Access the shared design system and style guide
- [ ] Review the design handoff process (see `docs/octoacme-design-handoff-checklist.md`)
- [ ] Schedule an introduction with the Product Manager to review user research findings

### Release Manager
- [ ] Review `docs/octoacme-release-checklist.md` and confirm ownership of all checklist items
- [ ] Confirm the current release calendar and next scheduled deployment with the Project Manager
- [ ] Meet with the QA Lead to align on release-readiness criteria

### Customer Support Lead
- [ ] Review the most recent post-release support summary and open user issues
- [ ] Align with the Product Manager on the process for submitting feedback to the backlog
- [ ] Confirm post-release support coverage with the Release Manager

### Technical Writer
- [ ] Review all existing documents in `docs/` for style, terminology, and consistency
- [ ] Identify any outdated or missing documentation based on the current project state
- [ ] Confirm the documentation delivery cadence with the Release Manager and Product Manager
