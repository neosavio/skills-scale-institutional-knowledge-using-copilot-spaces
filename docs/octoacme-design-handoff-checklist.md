# OctoAcme — Design Handoff Checklist

## Purpose
Ensure a smooth, complete handoff of design assets from the UX/UI Designer to the development team, reducing rework and implementation inconsistencies.

## When to Use
Complete this checklist before a design is considered ready for development. The UX/UI Designer and the receiving Developer should both sign off.

---

## Design Completeness

- [ ] All user stories / acceptance criteria covered by design mockups
- [ ] Desktop, tablet, and mobile breakpoints designed (or explicitly out of scope)
- [ ] Empty states, loading states, and error states included
- [ ] Edge cases (long text, missing data, extreme values) addressed in designs
- [ ] Accessibility requirements reviewed (colour contrast, focus states, ARIA labels noted)

## Assets & Specifications

- [ ] Final design files shared in agreed tool (e.g. Figma, Sketch) and link provided
- [ ] Assets exported in required formats (SVG, PNG, etc.) and placed in project asset folder
- [ ] Spacing, typography, and colour tokens documented or linked to style guide
- [ ] Interactive components and animations described with notes or prototype links
- [ ] Component names align with the existing design system / front-end component library

## Alignment & Review

- [ ] Product Manager has reviewed and approved the final designs
- [ ] Design has been reviewed against acceptance criteria with QA Lead
- [ ] Open design questions or known limitations documented in the design file and ticket
- [ ] Technical Writer informed of new UI patterns that require documentation updates

## Handoff Meeting (recommended for complex features)

- [ ] Walkthrough session scheduled with Developer(s) and QA Lead
- [ ] Developer questions captured and resolved before implementation begins
- [ ] Agreement on which design aspects are fixed vs. flexible during implementation

## Implementation Support

- [ ] UX/UI Designer available for questions during the development sprint
- [ ] Design review scheduled mid-sprint to validate implementation fidelity
- [ ] Final sign-off by UX/UI Designer before feature is considered "done"

---

## Contacts
| Role | Responsibility |
|------|---------------|
| UX/UI Designer | Prepares and delivers design assets; answers implementation questions |
| Product Manager | Approves final designs against product requirements |
| Developer | Implements designs and raises questions during development |
| QA Lead | Validates implementation against approved designs during testing |
| Technical Writer | Updates docs to reflect new UI patterns or workflows |
